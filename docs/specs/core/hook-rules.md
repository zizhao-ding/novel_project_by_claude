# Hook 检查规则 (Hook Rules)

> **优先级**: P0 - 必须遵守
> **最后更新**: 2026-06-25

## 概述

Hook 规则定义了在开发过程中，工具调用前后的自动检查机制，确保代码质量和规范一致性。

## PreToolUse (工具调用前)

在执行代码生成、文件操作等工具之前，必须检查以下规则：

### PRE-001: 文件路径验证

**检查内容**：
- 文件路径是否符合项目结构规范
- 文件命名是否符合命名规范
- 是否使用 `@` 别名导入

**检查规则**：
```typescript
// ✅ 正确路径
src/components/base/UserCard.vue
src/stores/novel.ts
src/services/auth.ts

// ❌ 错误路径
src/components/UserCard.vue          // 缺少子目录
src/Stores/novel.ts                  // 大写目录名
src/services/Auth.ts                 // 大写文件名
```

**处理方式**：
- 路径不符合规范 → 阻止执行，提示修正
- 路径符合规范 → 允许执行

### PRE-002: 代码规范预检查

**检查内容**：
- Vue 组件是否使用 `<script setup lang="ts">`
- 是否使用 Composition API
- Props 是否使用 TypeScript 接口定义
- 是否使用 `storeToRefs` 解构 store

**检查规则**：
```typescript
// ✅ 正确
<script setup lang="ts">
const { novels, loading } = storeToRefs(novelStore);

// ❌ 错误
<script lang="ts">
const { novels, loading } = novelStore;  // 缺少 storeToRefs
```

**处理方式**：
- 不符合规范 → 阻止执行，提示修正
- 符合规范 → 允许执行

### PRE-003: 依赖合法性检查

**检查内容**：
- 引入的依赖是否在 package.json 中
- 是否引入了禁止的依赖（如其他 UI 库）
- 是否使用了正确的导入方式

**检查规则**：
```typescript
// ✅ 正确
import { ElButton } from 'element-plus';
import { ref, computed } from 'vue';

// ❌ 错误
import { Button } from 'antd';          // 禁止其他 UI 库
import axios from 'axios';              // 应使用 api.client
```

**处理方式**：
- 依赖不合法 → 阻止执行，提示修正
- 依赖合法 → 允许执行

### PRE-004: 操作权限检查

**检查内容**：
- 是否执行了敏感操作（删除、覆盖等）
- 是否需要二次确认
- 是否在正确的分支上操作

**检查规则**：
```typescript
// ✅ 正确：敏感操作有确认
await ElMessageBox.confirm('确定删除吗？', '确认删除', { type: 'warning' });
await api.delete(id);

// ❌ 错误：直接执行敏感操作
await api.delete(id);  // 缺少确认
```

**处理方式**：
- 敏感操作无确认 → 阻止执行，提示添加确认
- 非敏感操作或有确认 → 允许执行

## PostToolUse (工具调用后)

在执行代码生成、文件操作等工具之后，必须检查以下规则：

### POST-001: ESLint 检查

**检查内容**：
- 代码是否符合 ESLint 规则
- 是否有语法错误
- 是否有未使用的变量

**检查方式**：
```bash
npm run lint
```

**处理方式**：
- 有 ESLint 错误 → 阻止提交，提示修正
- 无 ESLint 错误 → 允许提交

### POST-002: TypeScript 类型检查

**检查内容**：
- TypeScript 类型是否正确
- 是否有类型错误
- 是否有 `any` 类型

**检查方式**：
```bash
npm run type-check
```

**处理方式**：
- 有类型错误 → 阻止提交，提示修正
- 无类型错误 → 允许提交

### POST-003: 测试覆盖检查

**检查内容**：
- 新增代码是否有对应的测试
- 测试覆盖率是否达标
- 测试是否通过

**检查方式**：
```bash
npm run test:coverage
```

**处理方式**：
- 测试覆盖率不足 → 警告，建议补充测试
- 测试失败 → 阻止提交，提示修正
- 测试通过 → 允许提交

### POST-004: 文档同步检查

**检查内容**：
- 需求文档是否更新
- 验收标准是否勾选
- 技术实现要点是否更新
- **需求文档完整性检查**（新增）

**需求文档完整性检查**：

当创建或修改需求文档时，必须检查以下内容：

| 检查项 | 必填 | 说明 |
|--------|------|------|
| 概述 | ✅ | 一句话描述功能 |
| 用户故事 | ✅ | 作为 [角色]，我想要 [功能] |
| UI 布局 | ✅ | ASCII 示意图 |
| 路由 & 页面 | ✅ | 路由配置 |
| 组件树 | ✅ | 组件层级结构 |
| 数据流 & Store | ✅ | 状态管理设计 |
| API 契约 | ✅ | 接口定义 |
| 验收标准 | ✅ | Given-When-Then 场景 |
| 技术实现要点 | ✅ | 关键实现点 |
| **后端实现** | ✅ | 数据模型、API 实现、数据库设计 |
| 参考 & 备注 | ✅ | 相关文档链接 |

**后端实现章节检查**：

```markdown
## 10. 后端实现

### 数据模型
- [ ] 是否包含模型定义（Python SQLModel）
- [ ] 是否包含字段说明

### API 实现
- [ ] 是否列出所有端点
- [ ] 是否标注对应文件
- [ ] 是否标注实现状态

### 数据库设计
- [ ] 是否包含表名
- [ ] 是否包含字段列表
- [ ] 是否包含索引说明

### 依赖
- [ ] 是否列出后端依赖包
```

**检查方式**：
- 检查 `docs/requirements/` 中相关文档
- 检查验收标准 checkbox 状态
- **检查后端实现章节是否完整**

**处理方式**：
- 文档未更新 → 提醒更新文档
- 后端实现章节缺失 → 阻止提交，提示补充
- 文档已更新且完整 → 允许继续

### POST-005: 安全漏洞检查

**检查内容**：
- 是否有 XSS 风险
- 是否有 CSRF 风险
- 是否有敏感信息泄露

**检查规则**：
```typescript
// ✅ 正确：使用 Vue 的模板语法自动转义
<div>{{ userInput }}</div>

// ❌ 错误：直接插入 HTML
<div v-html="userInput"></div>  // XSS 风险
```

**处理方式**：
- 有安全漏洞 → 阻止提交，提示修正
- 无安全漏洞 → 允许提交
