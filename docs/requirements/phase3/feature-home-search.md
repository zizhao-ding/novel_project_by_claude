## 1. 概述

重构首页为内容发现页，承担平台"门面"角色。提供小说推荐、分类浏览、排行榜、搜索等核心分发能力。未登录用户可浏览首页和搜索结果，但需登录后才能进入阅读。

参考对象：起点中文网、番茄小说、微信读书的首页设计。

## 2. 用户故事

- 作为 **未登录访客**，我想要 **看到平台热门小说**，以便 **判断平台是否值得注册**
- 作为 **用户**，我想要 **在首页浏览推荐内容**，以便 **发现感兴趣的小说**
- 作为 **用户**，我想要 **按分类筛选小说**，以便 **找到特定题材的作品**
- 作为 **用户**，我想要 **搜索小说名称**，以便 **快速找到想看的小说**
- 作为 **用户**，我想要 **看到最近上新**，以便 **追更最新上传的小说**
- 作为 **用户**，我想要 **看到人气排行**，以便 **了解大家在看什么**

### 7.1 新增接口

### 7.2 接口定义

#### 7.2.1 热门推荐
```
GET /api/novels/hot?limit=10
说明：按加入书架数量降序排列（人气 = 收藏数）
Response: {
  code: 0,
  data: {
    items: [{
      id: number,
      title: string,
      file_size: number,
      category_id: number | null,
      category_name: string | null,
      bookshelf_count: number,    // 加入书架人数
      created_at: string
    }, ...]
  }
}
```

#### 7.2.2 最新上传
```
GET /api/novels/latest?limit=10&offset=0
说明：按上传时间降序排列
Response: 同热门推荐格式
```

#### 7.2.3 搜索
```
GET /api/novels/search?keyword=斗破&page=1&page_size=20
说明：根据小说标题模糊匹配
Response: {
  code: 0,
  data: {
    items: [...],     // 格式同 NovelListData
    total: number,
    page: number,
    page_size: number
  }
}
```

#### 7.2.4 增强现有 GET /api/novels
原接口增加参数：
```
GET /api/novels?page=1&page_size=20&category_id=1&sort=hot|latest|default
新增参数：
- category_id: 按分类筛选
- sort: 排序方式（default=按时间降序，hot=按书架数降序，latest=最新）
```

### 7.3 后端实现要点

#### 7.3.1 热门排行逻辑

- 统计每本小说被加入书架的次数（已有 bookshelf 表）
- 用 `SELECT novel_id, COUNT(*) as cnt FROM bookshelf GROUP BY novel_id ORDER BY cnt DESC`
- 联表查询 novel 信息返回

#### 7.3.2 搜索实现

- 数据库层面：`SELECT * FROM novels WHERE title LIKE '%keyword%'`
- 使用 SQLite 的 `LIKE` 即可，数据量小时无需全文索引
- 后续可升级为 FTS5 全文搜索

#### 7.3.3 权限

- 搜索和首页接口无需登录（公开访问）
- 认证装饰器 `get_current_user` 改为可选依赖
